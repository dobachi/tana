# Tana (棚) - 詳細表示＆ソート 設計メモ（バックログ）

> ステータス: 設計検討（未実装）。バックログ項目
> 最終更新: 2026-07-21
> 関連: 一覧描画は `src/js/core/filepane.js`、エントリは Rust `DirEntry`（`name/path/is_dir/size/is_hidden/modified`）

エクスプローラの「詳細表示」に相当する**列表示（更新日時など）**と、**クリック／キーボードでのソート**を実現するための方針。ベストプラクティス調査を含む。

---

## 1. 現状

- 一覧は `filepane.js` の `render()` が `<ul><li>` で描画。各行は **名前＋サイズ**の2要素のみ（`entry-name` / `entry-size`）。更新日時は未表示
- 並び順は **Rust `read_dir_entries` が固定**（フォルダ優先 → 名前の小文字昇順）。フロントで並べ替える仕組みは無い
- `DirEntry` に `modified`（更新時刻・エポック秒）は既に付与済み（プレビュー情報パネルで使用中）。サイズ・種別も取得済み

つまり「データは揃っている、UI とソート機構が無い」状態。

## 2. ベストプラクティス調査

### GUI ファイラ（マウス中心）
| 製品 | 詳細表示 | ソート操作 |
|---|---|---|
| Windows エクスプローラ | 名前/更新日時/種類/サイズ の列 | **列ヘッダクリックで昇順、再クリックで降順**。同キーで方向トグル |
| macOS Finder（リスト表示） | 同様の列 | 列ヘッダクリック、再クリックで方向反転。列ヘッダに ▲▼ |
| GNOME Files (Nautilus) | リスト表示で列 | ヘッダクリック。メニューからも選択可 |
| Total Commander | 常時列表示 | ヘッダクリック **＋ `Ctrl+F3`(名前)/`F4`(拡張子)/`F5`(日付)/`F6`(サイズ)** |

**共通点**: ①列ヘッダクリックで並べ替え、同じ列を再クリックで**方向トグル**、②現在のソート列と方向を**▲▼で明示**、③**フォルダ先頭**が既定。

### TUI ファイラ（キーボード中心）
| ツール | ソートキー |
|---|---|
| ranger | **`o` プレフィックス** + `n`(名前)/`s`(サイズ)/`m`(mtime)/`e`(拡張子)/`t`(型)、`r` で reverse。大文字 `O` で逆順メニュー |
| vifm | `:sort` コマンド＋ソートダイアログ（`set sort=...`） |
| lf | 設定で `sn`/`ss`/`st`（name/size/time）等を割当てるのが定番 |
| nnn | 単キーでソート種を循環（`t`=時刻トグル 等）、コンテキストで切替 |

**共通点**: **プレフィックス（`o`/`s`）＋種別キー**が主流。単キー循環は素早いが「今どの順か」を見失いやすい → **現在のソート状態の可視化が必須**。

### 名前ソートの原則（見落としがち）
- **自然順（natural / numeric-aware）**にする。`file2 < file10` となるよう、単純な文字列比較ではなく **`Intl.Collator(locale, { numeric: true, sensitivity: 'base' })`** を使う（日本語ロケールでも良好）。エクスプローラも自然順
- 既定は**大文字小文字を区別しない**

## 3. 設計方針（推奨）

### 3.1 データとソートは純粋関数に分離（真実源）

`core/sort.js`（純粋・テスト対象）:

```js
export const SORT_KEYS = ['name', 'size', 'modified', 'type', 'ext']; // 'type'=拡張子/種別
export const DEFAULT_SORT = { key: 'name', dir: 'asc', foldersFirst: true };

/** エントリ配列を (key, dir, foldersFirst) で安定ソートして新配列を返す。 */
export function sortEntries(entries, { key, dir, foldersFirst }, collator) { ... }

/** 同じキーなら方向トグル、違うキーなら asc にリセットした次の状態を返す。 */
export function nextSort(current, key) { ... }
```

- `collator` は `Intl.Collator(undefined, { numeric: true, sensitivity: 'base' })` を注入（テストでは固定ロケール）
- **安定ソート**（JS の `Array.prototype.sort` は安定）で、`size`/`modified` が同値のときは名前で tie-break
- `foldersFirst` が真ならフォルダ群を常に上に固める（GUI 既定）。設定で解除可能に
- **Rust 側の固定ソートはやめ、フロントで並べる**。一覧は既に取得済みなので**再ソートは I/O 不要で即時**（クリック連打でも軽い）。Rust は順序を保証しなくてよくなる

### 3.2 表示（詳細ビュー）

- `filepane.js` の各行を **名前 / サイズ / 更新日時** の3列に拡張（種別列は任意）。CSS Grid（`grid-template-columns`）で列を揃える。行は現状の `<li>` を活かしつつ内部を列化、または `role=row/gridcell`
- **列ヘッダ行**をペイン上部（`pane-header` の下）に追加：`名前 ▲ | サイズ | 更新日時`。現在のソート列に **▲/▼** を出す
- 更新日時の整形は既存 `render.js` の `formatMtime` を共通化（`core` へ移動して再利用）。相対表示（「3分前」等）は将来オプション
- **表示形式の切替**（簡易リスト ⇄ 詳細）を持つかは Open Issue（§5）。当面は**詳細を既定**にして単一表示でも良い（ファイラの標準）

### 3.3 操作（マウス＋キーボード両対応, NFR-U2）

**マウス**: 列ヘッダクリックでそのキーでソート、同じ列を再クリックで**方向トグル**（`nextSort`）。▲▼で明示。

**キーボード**: Tana はエディタ非搭載で単キーに余裕があるため、**GUI/TUI 双方の慣習を折衷**して:
- **`s` をソート・プレフィックス**にし、`s` に続けて `n`(name)/`s`(size)/`m`(modified)/`e`(ext) で並べ替え（ranger 流。覚えやすく衝突しない）
- 同じキーを再指定で方向トグル。`s` 単独＋`r` で現在キーの reverse
- 併せて**表示メニュー(V)に「並び替え ▸」サブメニュー**（名前/サイズ/更新日時 と 昇順/降順）を置き、ショートカットを併記（NFR-U6 ヘルプにも掲載）
- ※プレフィックス方式は「途中状態」を持つので、`s` 押下後はステータスバー等に「並び替え: 次のキー？」を短時間表示すると迷わない

（代替案: Total Commander 流の `Ctrl+F3/F5/F6`。発見性は高いがファンクションキーは押しにくい。Tana のキーボード指向には `s` プレフィックスが合う）

### 3.4 永続化

- ソート状態（key, dir, foldersFirst）を **localStorage/セッション(FR-14)** に保存。ペイン共通か個別かは Open Issue（§5）。既定は共通が単純
- 既定値: `name / asc / foldersFirst=true`

### 3.5 アクセシビリティ

- ヘッダは `<button>` にして `aria-sort="ascending|descending|none"` を付与。一覧は `role="grid"`、行は `role="row"`、セルは `role="gridcell"`

## 6. まとめ（結論）

- **列表示（名前/サイズ/更新日時）＋ 列ヘッダクリックで昇順→降順トグル＋ ▲▼ 明示** が GUI の王道。
- キーボードは **`s` プレフィックス（`sn/ss/sm/se`＋reverse）** が Tana のキーボード指向に最適。**表示メニューにも「並び替え」**を置き等価到達（NFR-U2/U6）。
- ソートは **`core/sort.js` の純粋関数**に切り出し、**Intl.Collator の自然順**・**フォルダ先頭**・**安定ソートで tie-break**。並べ替えは取得済みデータに対して**即時**（I/O 不要）。
- Rust の固定ソートは撤去し、順序はフロントが持つ。`DirEntry` は現状で足りる。
