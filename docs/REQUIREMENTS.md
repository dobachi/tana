# Tana (棚) - 要求分析

> 最終更新: 2026-06-08
> バージョン: 0.1.0 (ドラフト)
> 手法: ペルソナ/ゴール駆動 + FURPS+ 分類 + MoSCoW 優先度付け

---

## 1. 目的とスコープ

**Tana(棚)** は、ローカル・WSL・クラウド同期フォルダを横断して扱える、超軽量・ポータブルなクロスプラットフォーム ファイラである。
本書は要求を体系的に整理し、設計([DESIGN.md](DESIGN.md))の入力とする。

要求分析にあたっては、いただいた素の要望を以下の手法で体系化した。

- **ペルソナ/ゴール駆動**: 「誰が・なぜ使うか」から要求を導出
- **FURPS+ 分類**: 機能(F)/使いやすさ(U)/信頼性(R)/性能(P)/保守性(S)/+制約 で漏れを防ぐ
- **MoSCoW**: Must / Should / Could / Won't(今回) で優先度を明確化

## 5. 安全モードの要求詳細

ファイラは強力ゆえに「うっかり移動・削除」が事故になりやすい。Tana は **安全をデフォルト** とする。

| 項目 | 内容 |
|------|------|
| モード | **安全モード**(閲覧専用) / **操作モード**(変更可) の2状態 |
| 既定 | 起動時は**安全モード**(NFR-R1) |
| 切替 | ショートカットでトグル。切替は明示操作に限定し、偶発的な変更を防ぐ |
| 安全モードで抑止する操作 | 移動・コピー(書き込み先)・貼り付け・削除・リネーム・新規作成・上書き |
| 安全モードで許可する操作 | 閲覧・移動(カーソル)・プレビュー・検索・お気に入り参照・パスのコピー(クリップボードへの文字列) |
| 視覚表示 (NFR-U3) | ステータスバー/ウィンドウ枠の色とラベルでモードを明示(例: 安全=寒色, 操作=暖色) |
| 抑止時のフィードバック | 安全モード中に破壊的操作を試みた場合、トースト等で「安全モードです」と通知 |

> 設計上の決定事項(キー割当・配色・確認ダイアログの粒度)は [DESIGN.md](DESIGN.md) §安全モード設計 を参照。

## 6. 場所(Places)検出に関する要求

クラウド/WSL アクセスは「特別なAPI」ではなく「ファイルパス」で実現する(C-02)。各サービスは OS 上で次のように現れる。

| 対象 | OS 上の現れ方(例) |
|------|---------------------|
| OneDrive / OneDrive for Business | ユーザーフォルダ配下の同期フォルダ(`%USERPROFILE%\OneDrive` 等) |
| SharePoint | OneDrive 同期 or マップされたドライブ/ライブラリフォルダ |
| Box Drive | 仮想ドライブ / マウントされたフォルダ |
| WSL ディストロ | Windows からは `\\wsl$\<distro>\...`(UNCパス) |
| 標準フォルダ | ホーム・デスクトップ・ドキュメント・ダウンロード 等 |

要求: これらを起動時に検出し、サイドバーの「場所」一覧として提示する(FR-07)。検出に失敗しても、ユーザーが手動でパスを追加・お気に入り登録できること。

## 7. 今回対象外 (Won't / 将来拡張)

| 項目 | 理由 / メモ |
|------|------------|
| クラウドAPI直結(Microsoft Graph / Box API 等での未同期ファイル取得) | 認証・配布が大幅に重くなる。まずファイルパス方式で価値を出し、需要を見て検討 |
| ファイル内全文検索・インデックス | v1スコープ外。お気に入り検索(FR-06)とは別機能 |
| アーカイブ(zip 等)のインライン展開 | Could。プレビュー拡張の文脈で将来検討 |
| プラグイン機構 | 将来検討 |
| キーモード(vim/emacs風) | Fude 流用余地あり。Could |
